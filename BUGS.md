# Отчёт о дефектах API микросервиса объявлений

## BUG-001: Поле likes=0 трактуется как отсутствующее

| Поле | Значение |
|------|----------|
| **Краткое описание** | API возвращает 400 "поле likes обязательно" при передаче likes=0 в statistics, хотя 0 — валидное целое число |
| **Шаги воспроизведения** | 1. POST /api/1/item с телом: `{"sellerID": 123456, "name": "test", "price": 100, "statistics": {"likes": 0, "viewCount": 1, "contacts": 1}}` |
| **Фактический результат** | HTTP 400, `{"result":{"message":"поле likes обязательно","messages":{}},"status":"400"}` |
| **Ожидаемый результат** | HTTP 200, объявление создано (0 — допустимое значение для счётчика) |
| **Серьёзность** | Minor |
| **Окружение** | Host: https://qa-internship.avito.com |

## BUG-002: Поля viewCount и contacts=0 также отклоняются

| Поле | Значение |
|------|----------|
| **Краткое описание** | При likes=0 или contacts=0 API возвращает "поле X обязательно" |
| **Шаги воспроизведения** | POST с `statistics: {"likes": 1, "viewCount": 0, "contacts": 0}` |
| **Фактический результат** | HTTP 400 "поле contacts обязательно" |
| **Ожидаемый результат** | HTTP 200 |
| **Серьёзность** | Minor |
| **Окружение** | Host: https://qa-internship.avito.com |

## BUG-003: Формат ответа при создании объявления отличается от документации

| Поле | Значение |
|------|----------|
| **Краткое описание** | API возвращает `{"status": "Сохранили объявление - <uuid>"}` вместо полного объекта с id, name, price и т.д. |
| **Шаги воспроизведения** | 1. POST /api/1/item с валидным телом |
| **Фактический результат** | `{"status":"Сохранили объявление - 299400ad-8548-4b8c-a4e0-854f313abce2"}` |
| **Ожидаемый результат** | `{"id": "...", "sellerId": ..., "name": "...", "price": ..., "statistics": {...}, "createdAt": "..."}` согласно Postman |
| **Серьёзность** | Minor |
| **Окружение** | Host: https://qa-internship.avito.com |

## BUG-004: API принимает отрицательные price и sellerId

| Поле | Значение |
|------|----------|
| **Краткое описание** | API возвращает 200 при price=-100 и sellerId=-1, что нелогично для бизнес-логики |
| **Шаги воспроизведения** | POST с `price: -100` или `sellerID: -1` |
| **Фактический результат** | HTTP 200, объявление создано |
| **Ожидаемый результат** | HTTP 400, валидационная ошибка |
| **Серьёзность** | Medium |
| **Окружение** | Host: https://qa-internship.avito.com |

## BUG-005: Несуществующий ID возвращает 400 вместо 404

| Поле | Значение |
|------|----------|
| **Краткое описание** | Для невалидного формата ID возвращается 400 "ID айтема не UUID", для валидного несуществующего UUID — 400 или 404 в зависимости от эндпоинта |
| **Шаги воспроизведения** | GET /api/1/item/non-existent-uuid-12345 |
| **Фактический результат** | HTTP 400, "ID айтема не UUID" |
| **Ожидаемый результат** | HTTP 404 Not Found для несуществующего ресурса |
| **Серьёзность** | Low |
| **Окружение** | Host: https://qa-internship.avito.com |
