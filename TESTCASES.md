# Тест-кейсы для API микросервиса объявлений Avito QA Internship

**Host:** https://qa-internship.avito.com

**Эндпоинты:**
1. POST /api/1/item — Создать объявление
2. GET /api/1/item/{id} — Получить объявление по идентификатору
3. GET /api/1/{sellerId}/item — Получить все объявления продавца
4. GET /api/1/statistic/{id} — Получить статистику по объявлению

## 3. Corner cases (угловые случаи)

### TC-201: Идемпотентность создания
| Поле | Значение |
|------|----------|
| **Шаги** | 1. POST с одинаковыми данными (name, price, sellerId, statistics) 2 раза |
| **Ожидаемый результат** | Оба запроса HTTP 200, у объявлений разные id |
| **Техника** | Идемпотентность — повторный запрос создаёт новый объект |

### TC-202: Дублирование полей (name, price, sellerId)
| Поле | Значение |
|------|----------|

| **Техника** | Граничный случай — поля могут повторяться |
